08-Log关键字
  • 官方的Log注释
    • 原生代码中其实有大量的关键log的文档. 这类文档叫 event.logtags.
    • 其数据格式是: event|1|5
    • event表示含义;
    • 第一个数字表示数据类型, 包括:
      • 1: int
      • 2: long
      • 3: string
      • 4: list
      • 5: float
    • 第二个数字表示数据属性, 包括:
      • 1: Number of objects
      • 2: Number of bytes
      • 3: Number of milliseconds
      • 4: Number of allocations
      • 5: Id
      • 6: Percent
      • s: Number of seconds (monotonic time)
  • 定位应用线程ID, 过滤其Log
    • adb logcat -c
    • adb logcat | find "Start proc"
    • adb logcat | find "***" > 1.log
  • Input
    • 常规： Inputreader|inputdispatcher|eventhub|inputmanager_dispatch|InputTransport|PhoneWindowManager
    • View 处理触摸事件： processMotionEvent
      • Input log开关不打开时，这个打印也有，所以方便。
    • input rawdata： Input event
      • 其中，device即设备ID
      • Type, 事件类型.
        • 0，是EV_Syn，同步事件。因为一个事件要保上报多次，所以需要同步事件来确认整个事件上报完毕。
        • 1，是EV_key，键盘
        • 2，是EV_rel，即触摸事件。
      • code，指各种键值，key_***, syn_***等等。其中：
        • 116, 0x74, power键
        • 115, 0x73, 音量上
        • 114, 0x72, 音量下
      • value，指具体指值
        • 1 表示按下
        • 0 表示抬起
        • 触摸事件时，value表示坐标。x, y轴是分开上报的。
    • 事件上报： notifyMotion / notifyKey / dispatchMotion / dispatchKey
      • 分为两组，前者在入队列前打印，后两者在分发前打印。两组内容基本一样，仅policyFlags可能会发生变化。
      • 各枚举字段的含义在 input.h 中定义。
      • eventTime, 事件上报时间
      • downTime, down事件时间。
        • 注意，所有非down的事件，它的downTime不等于自己的eventTime，而是等于上一次的eventTime。
        • 凡是不满足这个条件的，都是非法事件，会被过滤掉。
      • deviceId： 设备id。即hal层input设备id。
        • R版本上，注入的事件统一设置为-1。
      • source，设备类型
        • 触屏： 0x1002
        • 键盘： 0x101
        • 鼠标： 0x2002
        • 手写笔： 0x4002
      • policyFlags： 即wms窗口策略。 重要
        • 1, 保持系统唤醒
        • 2, 是虚拟按键
        • 4, 是特殊功能修饰健（Ctrl、shift 之类）
        • 8,
        • 0x100 0000，注入的事件
        • 0x200 0000，可信的
        • 0x400 0000， 过滤的
        • 0x800 0000， 禁用自动重复事件。
        • 0x2000 0000，交互的事件。
        • 0x4000 0000，事件要传递到应用层，即 PASS_TO_USER.
      • action: 事件类型。重要。
        • 0： down
        • 1： up
        • 2： move
        • 3： cancel
        • 4： outside
        • 7： hover move
        • 9： hover enter
        • 10： hover exit
        • 11： button press
        • 12： button release
      • actionButton： 按键类型，比如左键、中键、右键之类。
      • flags：
        • 对于触摸事件，用于标记点位发生在屏幕边界，分上下左右。--不常用。
        • 对于按键事件，这个字段重要多了。
          • 2，软键盘产生的事件。
          • 8, 事件来自系统，不可被应用屏蔽。
      • Precision: 分x、y轴, 屏幕密度，一般是1。
      • keyCode: 按键键值
        • 在 Keycodes.h 定义。
        • 24, 0x18, 音量上
        • 25, 0x19, 音量下
        • 26, 0x1a, power键
      • scanCode: 硬件按键键值。底层上报的键值是这个。
        • 在 InputEventLabels.h 和 KeyEvent.java 中定义。
        • 116, 0x74, power键
        • 115, 0x73, 音量上
        • 114, 0x72, 音量下
      • metaState: 键盘，标识 Ctrl、ALT等按键的状态。 参见 ameta_none
    • 触摸屏报点： Pointer
      • id：点的序号，从0开始，一些input事件有多个点。
      • toolType： 触摸工具类型。
        • 1, 手指
        • 2, 手写笔
        • 3, 鼠标
        • 4, 橡皮擦 ？
        • 5, 手掌 ？
      • x, y： 点坐标
      • pressure: 按下压力值。用于按压屏或者手写笔。
      • size： 触摸区域的大小（近似值）。0到1之间。
      • touchMajor, tohchMinor, 触摸区域的长轴和短轴（单位为像素）
      • toolMajor, toolMinor, 触摸根工具的长轴和短轴 （单位为像素）
      • 这4个值一般一样。
      • orientation: 屏幕方向，0表示竖屏。
    • channel 发送事件的流程
      • enqueueDispatchCycle
      • prepareDispatchCycle
      • startDispatchCycle
      • publishMotionEvent
      • finishDispatchCycle
      • sendFinishedSignal
      • receiveFinishedSignal
    • 取得焦点窗口： Focus entered window
    • 事件cancel： InputDispatcher: cancel
      • 各字段含义和 notify Motion相同。
    • 事件cancel的原因：cancellation events
      • 其中，channel, cancel,发生的channel
      • synthesized，被cancel的事件的数量
      • mode，cancel模式
        • 1, 点事件
        • 2, 非点时间
        • 3, callback事件
    • 事件被跳过： skipping inconsistent motion event
    • 事件删除： Dropping
      • 第二个关键字是删除原因
    • 事件过滤： filter|shouldInterceptEventLocked
    • 事件冲突： Conflicting pointer actions
    • 事件注入： injectInputEvent|should inject
    • 事件拦截： interceptKeyBeforeQueueing|interceptKeyBeforeDispatching
    • 设备识别： Device added|New device|Input event
      • 其中，id 后面用得比较多。
      • classes 为设备类型，为掩码量。
    • 光标显示： unfade|fade|cursor
    • 横竖屏切换： Device reconfigured. 重要。
      • 其中，size： 屏幕坐标
      • orientation： 0表示竖屏, 1表示横屏。
      • mode： 设备模式，参见 DeviceMode.
