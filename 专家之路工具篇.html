专家之路工具篇 #工作需要
  • 编程基本配置
    • 字体：
      • Consolas（漂亮）
      • Source Code Pro（平实）
      • Courier New（干瘪）
    • 代码配色方案
      • 类: 棕色
      • 方法: 粉色
      • 成员变量: 金黄色
      • 局部变量: 紫色
      • 字符串: 绿色
      • 关键字: 蓝色
    • 字号： 20, 22, 24。 不能太小。
    • 键盘：机械键盘
    • 鼠标：整个好点的鼠标
  • 最佳工作方法
    • 本地工作代码管理
      • 本地放一套代码做完工作代码，写代码读代码过程中随时加注释。
      • 每周一，库上的代码备份一套。然后把new-old比较出来。然后合入最新代码，作为新的工作代码。
      • new-old 要每周备份。
      • as中导出注释，对比，确保不漏。
      • // dg1: 修改代码
      • // dg2: 类:  新增的注释
      • // dg3: 新增打印
      • 打印堆栈：new Exception(); // 多多使用。
    • 界面型问题，务必截图，一图胜千言。
    • 看代码，首先用一句话总结下每个类的功能，及类间关系。
    • 看文档时，关注主流程、多线程、互斥关系和定制项。
  • IDE
    • IDE: Intellij IDEA
      • Intellij IDEA 激活码 @iFile
    • IDE: Android Studio
      • AS获取代码的提交记录: 鼠标右键 - git - annotate --爽爆了
      • AS上的代码规范插件：
        • checkstyle
        • eclipce code formatter
      • Ctrl + click: 进入定义
      • Ctrl + alt + ← : 回退一步
      • Ctrl + alt + →: 前进一步
      • F11: 添加书签
      • shift + F11: 列出书签
      • double shift: 查找文件名
      • Ctrl + shift + F: 全局查找字符串
      • Ctrl + F12: 列出当前类的方法属性
      • Ctrl + E: 列出近期文件
      • Ctrl + alt + F7 / Ctrl + B: 查找应用处
      • Alt + Enter: 导入包
      • Ctrl + shift + Enter: 代码自动完成
      • Ctrl + 空格: 代码提示(和切换输入法重复了)
      • Ctrl + shift + 空格: 代码提示
      • F7: 步入
      • F8: 步过
      • shift + F8: 步出
      • Alt + shift + F7: 强制步入, 用于调试Python库代码
      • Alt + F9: 运行到光标处.
      • Alt + F8: 在调试过程中计算某个表达式的值
    • IDE: VSCode
      • 如何使用 Visual Studio Code 阅读 Android 源码 https://jekton.github.io/2018/05/11/how-to-read-android-source-code/
      • F1： 列出所有命令
      • cmd + P / cmd + E： 万能键. 当前界面是打开文件列表
        • 输入 >, 相当于F1, 列出所有命令
        • 输入 : , 相当于 cmd + G, 跳转到指定行
        • 输入 ?, 可以查看支持的所有命令
      • cmd + 点击： 跳转到定义
      • F12： 跳转到定义
      • 跳转到调用者
      • cmd + G： 跳转到指定行
      • cmd + Tab： 打开上一个文件
      • cmd + U： 返回上一个跳转位置
      • cmd + shift + F： 全局检索
      • Shift + F12： 所有引用的列表
      • cmd + B： 显示侧边栏
