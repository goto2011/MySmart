  • 最佳工作方法
    • 本地工作代码管理
      • 本地放一套代码作为工作代码，写代码读代码过程中随时加注释。
      • 每周一，工作代码备份一套。然后把new-old比较出来。然后合入最新代码，作为新的工作代码。
      • new-old 要每周备份。
      • as中导出注释，对比，确保不漏。
      • // dg1: 修改代码
      • // dg2: 新增注释
      • // dg3: 新增打印
    • 定位问题时，优先打印堆栈。
    • 界面型问题，务必截图，一图胜千言。
    • 看代码，首先用一句话总结下每个类的功能，及类间关系。
    • 看文档时，关注主流程、多线程、互斥关系和定制项。
  • 看代码: Source Insight
    • 硬伤在没有mac版, 以及部分c++11的语法不支持.
    • https://www.sourceinsight.com/trial/
    • #导入工程 : 新建工程 - 命名工程 - 设置源码目录, Project -> Synchronizing Files
    • #快捷键映射 : Options - Key Asssignments
    • #更换主题 : Visual Theme， 用 Dark Gold
    • #F1文件名查找 : File:Open
    • #F2符号查找 : View: Activate Project Symbol List
    • #F3上一处 : Navigation Go Back
    • #F4下一处 : Navigation Go Forward
    • #F5全文检索 : Symbol: Lookup References
      • keystring 查找最快
    • #F6关键字高亮 : View: Highlight Word
      • 选中单词后自动高亮： File Type Options - 勾选 Highlight references to selected symbol -- 有时候有点烦人, 还是关了的好.
    • #F7已打开文件列表: View:Activate Window List
    • #F8编辑区最大化 ：Symbol window
    • #F9书签列表: Navigation: Bookmark
    • #F10折叠代码 ： collapse all
    • #显示类继承关系 ：右击类名，选择“show in relation window”
    • #字体 ：options - file type options - screen font
    • 字号： Ctrl + 滑动鼠标滑轮
    • tab 键转空格： options - file type options - expand tabs to spaces
    • si上大括号自动缩进默认值是错的，选择 options - file type options - auto indent (右侧中间)， 取消 indent close brase.
    • 试用版过期的解决办法:
      • si 实在太贵, 盗版是说得过去的.
      • 用16进制编辑器（如sublime text）打开sourceinsight4.exe文件，找到c800 0000 742a 83bc 2408 这一段，修改74 为 eb。
      • 打开 C:\ProgramData\Source Insight\4.0\si4.lic
      • 将Expiration=”2017-XX-XX”中的2017修改为2030。
      • 启动si4, 会提示又可以使用31天了.
      • 过一段时间提示过期后，把Date="2019-10-24 00:00:00"，改成前一天的，又能继续使用。
